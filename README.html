Syntax & Formatting

  • two spaces indents, no tabs;
  • ideally, 80-characters wide lines;
  • properly written multi-line CSS rules;
  • meaningful use of whitespace.

Strings

Quotes
Using single quote 
// Yep
$direction: 'left';

// Nope
$direction: left;
String as CSS value
Specific CSS values (identifiers) such asinitialorsans-serifrequire not to be quoted.
URL
URLs are quoted as well

Numbers

Numbers shoud be display leading zeros before a decimal value less than one. Never display trailing zero 
// Yep
.foo {
  padding: 2em;
  opacity: 0.5;
}

// Nope
.foo {
  padding: 2.0em;
  opacity: .5;
}

List

either inlined or multilines;

Map

  • Space after colon
  • Open brace (() at the same line as the colon (:)
  • quoted keys iff they arre strings
  • each key/value pair on its own new line
  • comma (,) at the end of each key/value
  • trailing comma (,) on the last item.

Writting comments 
/**
 * Helper class to truncate and add ellipsis to a string too long for it to fit
 * on a single line.
 * 1. Prevent content from wrapping, forcing it on a single line.
 * 2. Add ellipsis at the end of the line.
 */
.ellipsis {
  white-space: nowrap; /* 1 */
  text-overflow: ellipsis; /* 2 */
  overflow: hidden;
}
